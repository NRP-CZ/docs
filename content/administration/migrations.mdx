# Migrations & Updates

<MigrationHeader
  date="2026-05-15"
  title="oarepo-app v4.0.0 separates CCMM dependencies"
  versionBadge="oarepo-app ≥ 4.0.0"
/>

Starting with `oarepo-app` version 4.0.0, CCMM dependencies are no longer included in the default production dependency set. Repositories that use CCMM must explicitly enable the `ccmm` extra.

<MigrationAffected>

| Affected | Not affected |
|----------|--------------|
| Repositories using CCMM and upgrading to `oarepo-app >= 4.0.0` | Repositories that do not use CCMM |
| Repositories with `oarepo-app[production]` only | Repositories already using `oarepo-app[production, ccmm]` |

</MigrationAffected>

<MigrationSteps>

**Update `pyproject.toml`**

In `pyproject.toml`, add the `ccmm` extra to the `oarepo-app` dependency if your repository uses CCMM:

**Before:**
```toml
dependencies = [
    "oarepo[s3,rdm]>=14.2.1b10.dev6,<15.0.0",
    "oarepo-app[production]==4.0.0",
]
```

**After:**
```toml
dependencies = [
    "oarepo[s3,rdm]>=14.2.1b10.dev6,<15.0.0",
    "oarepo-app[production, ccmm]==4.0.0",
]
```

After updating the dependency, run `./run.sh upgrade` to refresh the lock file.

</MigrationSteps>

<MigrationHeader
  date="2026-05-11"
  title="oarepo-app v2.3.0"
  prHref="https://github.com/oarepo/oarepo-app/pull/21"
  versionBadge="oarepo-app ≥ 2.3.0"
/>

This breaking change introduces the `oarepo-app` library in version 2.3.0. This release includes all Invenio changes released before 2026-05-01, including the introduction of CCMM vocabularies.

<MigrationAffected>

| Affected | Not affected |
|----------|--------------|
| Repositories created before 2026-05-11 | Repositories created after 2026-05-11 |
| Repositories using `oarepo-app <= 2.3.0`, or not using `oarepo-app` at all | --- |

</MigrationAffected>

<MigrationSteps>

**Update `pyproject.toml`**

1. In `pyproject.toml`, remove all `oarepo-*` dependencies and replace them with the new `oarepo-app` library. Do not remove the `oarepo[s3,rdm]` dependency.

  **Before:**
  ```toml
  dependencies = [
          "oarepo[s3,rdm]>=14.0.0,<14.1.9",
          "oarepo-runtime>=2.0.0dev0,<3.0.0",
          "oarepo-ui>=7.0.0dev0,<8.0.0",
          ...
  ]
  ```

  **After:**
  ```toml
  dependencies = [
      "oarepo[s3,rdm]>=14.2.1b10.dev6,<15.0.0",
      "oarepo-app[production]==2.3.0",  # <---- here
  ]
  ```

2. Update the version of the `oarepo` library:
  ```toml
  dependencies = [
      "oarepo[s3,rdm]>=14.2.1b10.dev6,<15.0.0",  # <---- here
      "oarepo-app[production]==2.3.0",
  ]
  ```
3. Make sure you use Python 3.14 in the `requires-python` section.
  ```toml
  requires-python = ">=3.14,<3.15"
  ```
4. Remove the top-level `package.json` if it exists.
5. Run `./run.sh upgrade` to update the lock file.
6. Run `./run.sh reset` to reset your local instance because of the CCMM vocabularies.
7. If you have sample data, you will need to update it to use CCMM vocabularies. See your `https://127.0.0.1:5000/vocabularies` which values are permitted.

   **For example:**
    1. Change the resource type ID from `dataset` to `c_ddb1`.
    2. Change the `created` time reference to `Created` (capitalized first letter).
    3. Change the `cc0-1.0` license to `CC0-1.0` (capitalized `CC`).

For inspiration, see [invenio-fzu PR #22](https://github.com/FZU-AV-CR/invenio-fzu/pull/22/changes?w=1)

<Callout type="warning">
If you are already running a production instance, please contact us first.
</Callout>

Make sure you have run the older migrations first.

</MigrationSteps>


<MigrationHeader
  date="2026-02-18"
  title="oarepo-ui v7.0+ Required Base Templates"
  prHref="https://github.com/oarepo/oarepo-ui/pull/409"
  versionBadge="oarepo-ui ≥ 7.0"
/>

oarepo-ui v7.0+ package requires base template files in your model's UI module. These templates must extend the oarepo-ui base templates. See [Record landing page documentation](/customize/model_ui/detail) for the complete template inheritance flow.

<MigrationAffected>

| Affected | Not affected |
|----------|--------------|
| Model UIs created before 2026-02-18 | Model UIs generated after 2026-02-18 |
| Using default templates | Custom page templates configured |
| **oarepo-ui >= 7.0** | older versions |

</MigrationAffected>

<MigrationSteps>

**Option 1: Use model update (Recommended)**

Run the model update command from your repository root:

```bash
./run.sh model update <model_name>
```

**Option 2: Create files manually**

If you prefer not to use model update, you can create the required template files manually in `ui/mymodel/templates/semantic-ui/mymodel/`. See the [model copier template](https://github.com/oarepo/nrp-model-copier/tree/main/template/ui/{%7B%7Bmodel_name%7D%7D}/templates/semantic-ui/{%7B%7Bmodel_name%7D%7D}) for the complete list of required files.

**Common issues:**

- `TemplateNotFound: mymodel/record_detail.html` → Create base template file
- Partials not showing → Check that `mymodel/record_detail/main.html` exist

</MigrationSteps>
