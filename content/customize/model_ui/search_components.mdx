# Search Result Components

This page documents the search result components for customizing record search displays. For deposit form components, see [Deposit form components](/customize/model_ui/deposit_components).

## Result Item Components

Search apps display results using list or grid layouts. The most commonly customized component is the result item.

### ResultsListItem

Displays individual records in search results. This component receives a `result` object containing the record metadata:

```jsx filename="ui/records/semantic-ui/js/Search/ResultsListItem.jsx"
export const ResultsListItem = ({ result }) => {
  return (
    <div className="result-item">
      <h3>{result.metadata.title}</h3>
      <p>{result.metadata.description}</p>
    </div>
  );
};
```

### ResultsGridItem

Similar to `ResultsListItem` but optimized for grid layout views.

The result object includes the full record metadata, allowing you to display any fields from your data model.

## Initializing Search App with Custom Components

Create a search app entry point that registers your custom components:

```jsx filename="ui/records/semantic-ui/js/Search/index.js"
import {
  parseSearchAppConfigs,
  createSearchAppsInit,
  SearchAppFacets,
} from "@js/oarepo_ui/search";
import { parametrize } from "react-overridable";
import ResultsListItem from "./ResultsListItem";

// Customize facets with additional props
const SearchAppFacetsWithProps = parametrize(SearchAppFacets, {
  allVersionsToggle: true,
});

const [{ overridableIdPrefix }] = parseSearchAppConfigs();

export const componentOverrides = {
  [`${overridableIdPrefix}.SearchApp.facets`]: SearchAppFacetsWithProps,
  [`${overridableIdPrefix}.ResultsList.item`]: ResultsListItem,
};

createSearchAppsInit({ componentOverrides });
```

## Overridable IDs

Find available component IDs in:

<GitHubCode repo="oarepo/oarepo-ui" path="oarepo_ui/theme/assets/semantic-ui/js/oarepo_ui/search/constants.js" />

The overridable ID format follows:
```
{applicationId}.Search.{componentId}
```

For example, for a record model named "documents":
- `Documents.Search.ResultsList.item`
- `Documents.Search.SearchApp.facets`
- `Documents.Search.SearchApp.sort`

See [Search result](/customize/model_ui/search) for more information on search applications.
